# 做饭灵感 App - 产品需求文档 (PRD)

## 1. 产品概述

### 1.1 产品定位
"做饭灵感"是一款针对家常烹饪的场景化菜谱推荐应用，帮助用户根据现有食材快速获取烹饪灵感。

### 1.2 目标用户
- 厨艺有限的年轻人
- 希望利用冰箱剩余食材的家庭主妇/夫
- 追求快手家常菜的上班族

### 1.3 核心价值主张
"拍照识别食材，AI推荐菜谱，让做饭不再纠结"

---

## 2. 功能需求

### 2.1 核心功能

#### 2.1.1 冰箱管理（食材录入）
- **拍照上传**：用户拍摄或从相册选择食材照片
- **AI视觉识别**：自动识别图片中的食材种类
- **手动管理**：支持添加、删除、编辑食材名称
- **数据持久化**：食材列表存储于 LocalStorage

#### 2.1.2 菜谱推荐
- **智能匹配**：根据用户现有食材推荐可做的菜谱
- **匹配度计算**：基于主食材匹配度排序
- **筛选功能**：按烹饪方式（炒/煮/蒸/烤/炸/凉拌）筛选

#### 2.1.3 菜谱详情
- **主食材展示**：列出菜谱所需的主食材
- **调料清单**：列出所需调料与辅料
- **步骤说明**：详细的烹饪步骤
- **小贴士**：烹饪技巧提示

### 2.2 用户交互流程

```
首页（冰箱） → 拍照/添加食材 → AI识别食材
    ↓
推荐列表页 → 查看匹配菜谱
    ↓
菜谱详情页 → 查看主食材/调料/步骤
    ↓
开始烹饪
```

## 3. 菜谱推荐逻辑

### 3.1 推荐算法

#### 3.1.1 食材匹配
- 用户食材与菜谱主食材进行智能匹配
- 支持食材别名（如"番茄"="西红柿"）
- 匹配时不区分大小写和前后空格

#### 3.1.2 匹配度计算
```
匹配度 = 已匹配的主食材数 / 菜谱所需主食材总数
```

#### 3.1.3 主辅料分离机制
- **主食材**：决定菜品属性的核心食材（如番茄、鸡蛋、五花肉）
- **调料类**：油、盐、酱油、醋、葱、姜、蒜等常备调味品
- **重要**：匹配度仅根据主食材计算，调料类不计入匹配度分母
- 主食材100%匹配时，视为"食材已备齐"

### 3.2 推荐排序规则
1. **完全匹配优先**：主食材全部在库 > 部分匹配
2. **匹配度高优先**：匹配度分数高者优先
3. **缺少少优先**：缺少主食材种类少者优先
4. **时间短优先**：烹饪时间短者优先

## 4. 界面原型说明

### 4.1 页面结构

| 页面 | 功能 | 路径 |
|------|------|------|
| 首页/冰箱 | 展示已识别食材，支持拍照上传 | `/` |
| 推荐列表 | 展示匹配菜谱，支持筛选排序 | `/recommend` |
| 菜谱详情 | 展示完整菜谱信息 | `/recipe/[id]` |
| 收藏页 | 展示收藏的菜谱 | `/favorites` |
| 我的 | 用户信息和设置 | `/profile` |

### 4.2 底部导航
- 首页（冰箱图标）
- 推荐（灯泡图标）
- 收藏（心形图标）
- 我的（用户图标）

## 5. 技术约束

### 5.1 平台要求
- iOS Safari 最新版
- 移动端优先（Mobile-first）
- 最小点击区域 44px
- 安全区域适配（safe-area）

### 5.2 AI模型
- 视觉识别：阿里云通义千问 qwen-vl-plus
- 文本推荐：阿里云通义千问 qwen-plus
- API兼容模式：OpenAI SDK

### 5.3 数据存储
- 前端状态：Zustand
- 服务端状态：TanStack Query
- 持久化：LocalStorage
- 云端（待开发）：Firebase

---

## 6. 非功能需求

### 6.1 性能要求
- 页面首次加载 < 3秒
- 图片压缩至 1024px 长边
- API 请求超时 30秒

### 6.2 错误处理
- API 错误必须展示给用户
- 不使用静默兜底假数据
- 终端日志完整追踪
