# 小锅灵感 App V2.0 实施摘要

更新时间：2026-05-26

本文档记录 V2.0 当前已实现的核心功能、数据结构、推荐规则与验收状态。它用于补充 PRD 和开发进度文档，不替代正式 PRD。

## 1. 产品范围

V2.0 在 MVP 基础上升级为三 Tab 结构：

- 冰箱：管理当前食材、拍照/相册识别、手动添加、删除、标星、本餐偏好、生成今日灵感。
- 灵感：展示推荐菜谱、分类筛选、搜索、刷新、加载更多、进入详情页。
- 我：本地头像昵称、专属口味档案、版本号、隐私政策和用户服务协议入口。

本期不做真实登录，不做云端同步，不接 Supabase/Firebase 账号体系。头像、昵称、饮食偏好均为本地个性化数据。

## 2. 本地数据层

移动端使用 AsyncStorage 保存以下数据：

- 食材列表：包含 `id`、`name`、`category`、`isPinned`。
- 本餐偏好：`speed`、`comfort`、`light`、`protein`、`lessOil`。
- 用户偏好：饮食目标、口味偏好、忌口与禁忌、首次引导完成状态。
- 本地资料：昵称、头像 URI。
- 匿名设备 ID：本地生成，随 API Header 发送。
- 推荐缓存：根据推荐上下文生成 Hash。偏好保存后会清理缓存并写入刷新标记，灵感页重新聚焦时自动刷新。
- 菜谱详情缓存：缓存已生成详情。新生成详情在写入缓存前完成主食材/调料归类和用量估算。

## 3. 默认偏好标签

Onboarding 和“我”页共用同一套偏好编辑器。

饮食目标：

- 认真吃好每一顿
- 轻盈减脂
- 清淡养生
- 高蛋白增肌

口味偏好：

- 清淡鲜美
- 经典下饭
- 咸甜交织
- 无辣不欢
- 酸甜开胃
- 轻食西餐

忌口与禁忌：

- 不吃香菜
- 不吃葱

偏好编辑器只展示当前默认标签和用户手动添加的自定义标签。已废弃的旧默认标签不会继续作为可选项展示。

- 右上角编辑按钮。进入编辑模式后，食材标签显示删除入口。

- CTA 文案为“解锁今日做饭灵感”。

## 5. 灵感页

已实现：

- 自动加载推荐缓存或生成新推荐。
- 从冰箱 CTA 携带 `generate` 参数进入时强制生成。
- 偏好保存后清缓存，并在灵感页重新聚焦时自动刷新。
- 分类筛选：全部、炒、煮、蒸、烤、凉拌。
- 搜索菜谱名称。

## 6. 推荐 API

`POST /api/recommend` 接收完整推荐上下文：

- `ingredients`
- `pinnedIngredients`
- `mealPreference`
- `userPreferences`
- `excludeRecipeTitles`
- `pageSize`

服务端使用模型生成结构化 JSON，并做确定性后处理。

排序规则：

1. 匹配度高的优先。
2. 同匹配度下，包含标星食材的优先。
3. 再按策略 `A > B > C` 排序。
4. 再按缺少主食材更少排序。
5. 最后按烹饪时间更短排序。

策略定义：

- A：标星食材 + 本餐偏好 + 长期偏好。
- B：标星食材 + 本餐偏好。
- C：基于当前食材发散。

## 7. 推荐生成规则

已确认并实现的规则：

- 匹配度只计算主食材，不计算调料。
- 标星食材是本次明确意图，可以突破“同一主食材最多出现 2 次”的多样性限制。
- 通用肉类可以匹配常规切配形态，例如猪肉、肉片、肉丝、肉末、五花肉、里脊肉。
- 通用肉类不能匹配特殊部位，例如排骨、猪肝、猪蹄、牛腩、牛腱、牛排、鸡翅、鸡爪。
- 菜名里的主食材必须出现在主食材数组里。
- 菜名应接近真实家常菜名，避免半截菜名或机械堆叠，例如避免“猪肉黄瓜炒”这类不完整表达。
- 多选口味偏好应尽量在推荐列表中覆盖，也允许合理组合，例如偏辣菜、西餐菜、辣味西式菜都可以出现。
- 组合偏好不能牺牲食材匹配、真实菜名和正常做饭逻辑。
- `意大利面`、`意面`、`意粉`、`通心粉` 互为别名。

主食材与调料边界：

- 主食材：肉、蛋、豆制品、蔬菜、主食等决定菜品主体的食材。
- 调料与辅料：油、盐、糖、醋、生抽、老抽、料酒、蚝油、淀粉、胡椒粉、辣椒粉、豆瓣酱、番茄酱、黄豆酱、甜面酱、葱姜蒜、清水、高汤等。
- 番茄酱不是番茄，不能进入主食材，也不能按“半个”估量。

## 8. 菜谱详情页

已实现：

- 调用 `/api/detail` 生成步骤和小贴士。
- 主食材和调料分区展示。
- 主食材用量做本地估算，例如肉类 150 克、鸡蛋 2 个、意大利面 150 克。
- 调料用量做本地估算，例如番茄酱 1 汤勺、盐 1 小勺。
- 步骤中主食材和调料统一用橙色高亮。
- 详情失败展示友好错误和重试按钮。
- 限流错误展示“大厨需要喝口水休息一下”语义。

## 9. 我的页

已实现：

- 顶部头像和昵称。
- 支持本地选择头像。
- 支持本地编辑昵称。
- “专属口味档案”模块。
- 保存成功使用轻量灰白 Toast。
- 保存偏好后自动影响下一次推荐，并触发灵感页刷新。
- 隐私政策和用户服务协议入口保留，待正式文档接入。

隐私政策建议说明：

- 昵称、头像、饮食目标、口味偏好、忌口等当前仅保存在本机。
- 当前资料不关联真实账号，不做云端同步。
- 后续如果接入账号或云同步，需要更新隐私政策并获得用户知情。

用户服务协议建议说明：

- 当前账号资料为本地个性化展示，不构成真实登录账号。
- 推荐内容为辅助参考，用户仍需自行判断食材安全、过敏风险和烹饪可行性。
