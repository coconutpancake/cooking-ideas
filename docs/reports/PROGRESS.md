# 小锅灵感 App - 开发进度

## 当前状态

小锅灵感 V2.0 核心功能已完成，当前主线是 Expo 移动端 + Next.js API 的 Monorepo 架构。

- 分支：`v2-dev`
- 移动端：`apps/mobile-app`
- API 服务：`apps/web-api`
- 文档：`docs`

本期不做真实登录、不做云端同步、不接入 Firebase/Supabase 账号体系。头像、昵称、偏好、设备 ID、推荐缓存都只保存在本机。

## 当前技术架构

- 移动端：Expo Router、React Native、NativeWind、AsyncStorage
- API：Next.js Route Handler、TypeScript、OpenAI SDK 兼容调用
- AI 模型：通义千问文本模型与视觉模型，通过环境变量配置
- 本地存储：食材、偏好、本地资料、设备 ID、推荐缓存、详情缓存
- 后端 AI 服务层：`ai.service.ts` 保持对外薄入口；视觉识别、推荐编排、推荐 prompt、推荐解析、推荐匹配/归一化分别拆分维护。
- 移动端偏好层：偏好类型保留在 `lib/types.ts`，默认标签和旧标签过滤规则维护在 `lib/preferences.ts`，UI 组件只负责展示和交互。

## API 路由

- `POST /api/vision`：识别图片中的食材
- `POST /api/recommend`：基于食材、标星食材、本餐偏好、长期偏好生成推荐
- `POST /api/detail`：生成菜谱详情、用量和步骤

## 推荐规则

推荐输入：

- 全部食材
- 标星食材
- 本餐偏好
- 长期偏好
- 已展示菜名
- pageSize

推荐排序：

1. 匹配度优先
2. 包含标星食材优先
3. 策略 A > B > C
4. 缺少主食材更少优先
5. 烹饪时间更短优先

核心约束：

- 标星食材是本次推荐优先条件，可突破同一主食材最多出现两次的限制。
- 通用肉类只和常规切配形态互相匹配，不等同于具体部位。
- 主食材只放决定菜品主体的食材，如肉、蛋、豆制品、蔬菜、主食。
- 油、盐、糖、醋、生抽、老抽、料酒、蚝油、淀粉、胡椒粉、辣椒粉、豆瓣酱、番茄酱、葱姜蒜、清水、高汤等属于调料或辅料。
- 多选口味偏好应在推荐列表中尽量覆盖，也允许合理组合，但不能牺牲真实菜名、食材匹配和正常做法。
